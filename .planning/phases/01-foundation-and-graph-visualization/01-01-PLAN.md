  <files>
    frontend/package.json, frontend/tsconfig.json, frontend/tsconfig.app.json,
    frontend/vite.config.ts, frontend/tailwind.config.js, frontend/postcss.config.js,

    frontend/src/lib/utils.ts
  </files>
  <action>
    1. Run `npm create vite@latest frontend -- --template react-ts` from the project root
    2. `cd frontend && npm install`
    3. Install core dependencies:
       ```
       npm install react-force-graph-2d @tanstack/react-query @tanstack/react-table zustand react-router-dom lucide-react clsx tailwind-merge class-variance-authority @radix-ui/react-slot
       ```
    4. Install Tailwind v3 dev dependencies:
       ```
       npm install -D tailwindcss@^3.4.0 postcss autoprefixer tailwindcss-animate
       npx tailwindcss init -p
       ```
    5. Configure tailwind.config.js:
       - Set `darkMode: "class"`
       - Set `content: ["./index.html", "./src/**/*.{ts,tsx}"]`
       - Extend theme with shadcn CSS variable colors (background, foreground, primary, secondary, muted, accent, destructive, card, popover, border, input, ring)
       - Add tailwindcss-animate to plugins
    6. Update src/index.css with Tailwind directives:
       ```css
       @tailwind base;
       @tailwind components;
       @tailwind utilities;
       ```
       Add CSS custom properties for shadcn theme (both :root for light and .dark for dark mode). Use colors matching a clean developer-focused aesthetic:
       - Light: near-white backgrounds, dark text, indigo-tinted accents
       - Dark: dark gray backgrounds (#1a1a2e area), light text, same accent palette
    7. Create src/lib/utils.ts with the cn() utility:
       ```typescript
       import { type ClassValue, clsx } from "clsx";
       import { twMerge } from "tailwind-merge";
       export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)); }
       ```
    8. Configure vite.config.ts with:
       - @vitejs/plugin-react
       - Path alias: `@` -> `./src`
       - Proxy `/api` to `http://localhost:8080` for dev server
       - Build rollupOptions.output.manualChunks to separate vendor chunks (react, react-dom, react-force-graph-2d, tanstack)
    9. Update tsconfig.app.json: add `"baseUrl": ".", "paths": {"@/*": ["./src/*"]}` to compilerOptions. Set `verbatimModuleSyntax: false`.
    10. Initialize shadcn with `npx shadcn@2.3.0 init` (select: New York style, Tailwind v3, configure components.json)
    11. Install shadcn components:
        ```
        npx shadcn@2.3.0 add button sheet dialog input table badge
        ```
    12. Update src/main.tsx to render App with React.StrictMode
    13. Update src/App.tsx to render a placeholder "OpenGraphDB" heading to verify setup
  </action>

  <action>
    1. Create frontend/src/types/graph.ts with:
       ```typescript
       export interface GraphNode {
         id: string | number;
         labels: string[];
         properties: Record<string, unknown>;
         label?: string;        // Display label (first label or id)
         x?: number;
         y?: number;
         fx?: number | null;    // Fixed position (drag)
         fy?: number | null;
         __bckgDimensions?: [number, number]; // Used by canvas renderer
       }

       export interface GraphEdge {
         id: string | number;
         source: string | number | GraphNode;
         target: string | number | GraphNode;
         type: string;          // Relationship type
         properties: Record<string, unknown>;
       }

       export interface GraphData {
         nodes: GraphNode[];
         links: GraphEdge[];
       }

       export type ViewMode = 'graph' | 'table';
       ```

    2. Create frontend/src/types/api.ts with:
       ```typescript
       export interface HealthStatus {
         connected: boolean;
       }

       export interface QueryResponse {
         nodes: Array<{
           id: string | number;
           labels: string[];
           properties: Record<string, unknown>;
         }>;
         relationships: Array<{
           id: string | number;
           type: string;
           startNode: string | number;
           endNode: string | number;
           properties: Record<string, unknown>;
         }>;
         columns?: string[];
         rows?: unknown[][];
       }

       export class ApiError extends Error {
         constructor(
           message: string,
           public status: number,
           public details?: unknown
         ) {
           super(message);
           this.name = 'ApiError';
         }
       }
       ```

       - Apply bg-background text-foreground for theme support
