**Step 2: Create `frontend/src/lib/ai/providers.ts`**

Define the core ChatProvider interface and supporting types:

```typescript
export interface ChatMessage {
  role: 'user' | 'assistant' | 'system'
  content: string
}

export interface ChatProvider {
  id: string
  label: string
  isAvailable: () => boolean | Promise<boolean>
  init?: (opts?: { onProgress?: (msg: string, pct: number) => void }) => Promise<void>
  streamChat: (args: {
    messages: ChatMessage[]
    signal?: AbortSignal
    onChunk: (text: string, done: boolean) => void
  }) => Promise<void>
  dispose?: () => Promise<void>
}

export type AIProviderType = 'webllm' | 'openai' | 'anthropic' | 'gemini' | 'openai-compatible'
```

Add `PROVIDER_MODELS` constant: a record keyed by `AIProviderType` with an array of `{ id: string; label: string }` objects for the curated model list:
- webllm: `[{ id: 'Qwen2.5-1.5B-Instruct-q4f16_1-MLC', label: 'Qwen 2.5 1.5B (Free, Local)' }]`
- openai: `[{ id: 'gpt-4o', label: 'GPT-4o' }, { id: 'gpt-4o-mini', label: 'GPT-4o Mini' }]`
- anthropic: `[{ id: 'claude-sonnet-4-20250514', label: 'Claude Sonnet 4' }, { id: 'claude-haiku-4-20250414', label: 'Claude Haiku 4' }]`
- gemini: `[{ id: 'gemini-2.0-flash', label: 'Gemini 2.0 Flash' }, { id: 'gemini-1.5-pro', label: 'Gemini 1.5 Pro' }]`
- 'openai-compatible': `[]` (user provides custom model name)

Add `createProvider` factory function:
```typescript
export function createProvider(
  type: AIProviderType,
  config: { apiKey: string; model: string; baseUrl: string }
): ChatProvider
```
This function imports and instantiates the appropriate provider adapter based on type. Use dynamic imports (lazy) for each provider to avoid bundling all SDKs upfront.

**Step 3: Create `frontend/src/lib/ai/webllm-provider.ts`**

Implement `WebLLMProvider` class implementing `ChatProvider`:
- `id: 'webllm'`, `label: 'WebLLM (Free, Local)'`
- `isAvailable()`: returns `'gpu' in navigator` (WebGPU detection)
- `init(opts)`: Creates `MLCEngine` via `CreateMLCEngine` from `@mlc-ai/web-llm` with model `Qwen2.5-1.5B-Instruct-q4f16_1-MLC`. Pass `initProgressCallback` that calls `opts.onProgress`. Hold engine in a module-level variable (singleton) so re-init checks `engine !== null` first.
- `streamChat({ messages, signal, onChunk })`: Call `engine.chat.completions.create({ messages, stream: true })`. Iterate with `for await (const chunk of stream)`, extract `chunk.choices[0]?.delta.content ?? ''`, call `onChunk(token, false)`. After loop, call `onChunk('', true)`. Respect `signal` by checking `signal?.aborted` inside the loop and breaking if true.
- `dispose()`: call `engine.unload()` if engine exists, set engine to null.

**Step 4: Create `frontend/src/lib/ai/openai-provider.ts`**

Implement `OpenAIProvider` class implementing `ChatProvider`:
- Constructor takes `{ apiKey, model, baseUrl? }`.
- `id: 'openai'` (or `'openai-compatible'` if baseUrl provided), `label` derived from constructor.
- `isAvailable()`: returns `true` (always available if API key present).
- `streamChat({ messages, signal, onChunk })`: Create `OpenAI` client with `{ apiKey, dangerouslyAllowBrowser: true }` and optional `baseURL` if provided. Call `client.chat.completions.create({ model, messages, stream: true, max_tokens: 2048 })`. Iterate async chunks, extract `chunk.choices[0]?.delta?.content ?? ''`, call `onChunk(token, false)`. After, `onChunk('', true)`. Pass `signal` if the SDK supports it via fetch options.

**Step 5: Create `frontend/src/lib/ai/anthropic-provider.ts`**

Implement `AnthropicProvider` class implementing `ChatProvider`:
- Constructor takes `{ apiKey, model }`.
- `id: 'anthropic'`, `label: 'Anthropic'`.
- `isAvailable()`: returns `true`.
- `streamChat({ messages, signal, onChunk })`:
  - Create `Anthropic` client with `{ apiKey, dangerouslyAllowBrowser: true }`.
  - Separate system message from conversation messages (Anthropic API takes `system` as a separate parameter, not in messages array).
  - Call `client.messages.stream({ model, max_tokens: 2048, system: systemContent, messages: nonSystemMessages })`.
  - Iterate: `for await (const event of stream)`, check `event.type === 'content_block_delta' && event.delta.type === 'text_delta'`, call `onChunk(event.delta.text, false)`.
  - After loop: `onChunk('', true)`.

**Step 6: Create `frontend/src/lib/ai/gemini-provider.ts`**

Implement `GeminiProvider` class implementing `ChatProvider`:
- Constructor takes `{ apiKey, model }`.
- `id: 'gemini'`, `label: 'Google Gemini'`.
- `isAvailable()`: returns `true`.
- `streamChat({ messages, signal, onChunk })`:
  - Create `GoogleGenAI` from `@google/genai` with `{ apiKey }`.
  - Convert ChatMessage[] to Gemini format: system message goes to `config.systemInstruction`, user/assistant messages go to `contents` array with `role: 'user'` / `role: 'model'`.
  - Call `ai.models.generateContentStream({ model, contents, config: { systemInstruction, maxOutputTokens: 2048 } })`.
  - Iterate: `for await (const chunk of response)`, extract `chunk.text ?? ''`, call `onChunk(token, false)`.
  - After: `onChunk('', true)`.

**Step 7: Create `frontend/src/lib/ai/system-prompt.ts`**

Export `buildSystemPrompt(schema: SchemaResponse): string` function:
- Takes the schema response from `useSchemaQuery()`.
- Returns a system prompt string that includes:
  1. Role description: "You are an expert Cypher query generator for a graph database."
  2. Schema context: node labels, relationship types, property keys serialized as comma-separated lists.
  3. Rules: generate valid openCypher, use LIMIT 100 by default, explain before showing query, wrap Cypher in triple-backtick `cypher` code blocks, explain errors and provide corrected queries.
  4. Two few-shot examples: one simple MATCH (show all movies), one traversal (find actors who worked with a named actor). Use generic labels since the actual schema is injected.
- Also export `extractCypherBlocks(markdown: string): string[]` that uses regex `/ ```(?:cypher|CYPHER)\n([\s\S]*?)``` /g` to extract Cypher code from AI responses. Only call this on finalized (non-streaming) messages.
- Also export `buildResultSummary(nodeCount: number, rowCount: number, sampleProps: string): string` that produces a short summary for feeding back to the AI after query execution.

**IMPORTANT:** Do NOT use `@google/generative-ai` (deprecated). Use `@google/genai` (GA May 2025).
