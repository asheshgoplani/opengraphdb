**Implementation details:**

1. **Provider initialization**: Use `useRef` to hold the current `ChatProvider` instance. Use `useEffect` to create/recreate the provider when settings change (aiProvider, aiApiKey, aiModel, aiBaseUrl from `useSettingsStore`). Call `createProvider(type, config)`.

2. **WebLLM lazy init**: If provider is WebLLM, do NOT initialize on mount. Instead, init on first `sendMessage` call. During init, call `provider.init({ onProgress })` and pipe progress to `setDownloadProgress`. After init completes, set `downloadProgress` to null. If `!provider.isAvailable()`, show a message in chat: "Your browser doesn't support local AI. Configure an API key in Settings to use the assistant." (This is the WebGPU fallback per CONTEXT.md).

3. **Provider priority**: When `aiApiKey` is non-empty and `aiProvider !== 'webllm'`, use the configured provider. When `aiApiKey` is empty, fall back to WebLLM. This matches the decision: "API key takes priority. WebLLM is the fallback."

4. **Schema context**: Use `useSchemaQuery()` to get schema data. Build system prompt via `buildSystemPrompt(schema)`. The system message is prepended to every request.

5. **Message rolling window**: To prevent context window overflow (Pitfall 3 in research), keep only the system prompt + last 6 message exchanges (12 messages). For WebLLM (4096 token limit), keep last 4 exchanges (8 messages).

6. **sendMessage(text: string)** function:
   - Call `addUserMessage(text)` on the chat store.
   - Set `isLoading(true)`.
   - If WebLLM provider needs init, trigger init with progress callbacks.
   - Build messages array: system prompt + rolling window of recent messages.
   - Create an `AbortController` for cancellation.
   - Call `startAssistantMessage()` to get assistant message id.
   - Call `provider.streamChat({ messages, signal, onChunk: (token, done) => { ... } })`.
     - In `onChunk`: if not done, call `appendToMessage(id, token)`. If done, call `finalizeMessage(id, extractCypherBlocks(fullContent))`.
   - On error: append an error message to the chat.
   - Set `isLoading(false)` in finally.

7. **runCypherFromAI(cypher: string, messageId: string)** function:
   - This is called when the user clicks "Run query" on an AI-generated Cypher block.
   - First, call `clearTrace()` from graph store (per Pitfall 5: clear before new trace).
   - Execute via `useTraceQuery.mutateAsync({ cypher, onTraceStep })` where `onTraceStep` calls `advanceTrace`.
   - On success: build a result summary via `buildResultSummary(...)`, call `setQueryResult(messageId, summary)`, then automatically send the summary back to the AI as a follow-up (per CONTEXT.md: "feed a summary of results back to the AI so it can provide follow-up analysis"). Call `sendMessage` internally with a system-injected result summary.
   - On error: call `setQueryError(messageId, errorMessage)`, then automatically send the error back to the AI for correction (per CONTEXT.md: "the AI automatically receives that feedback and responds with an explanation and revised query"). Call `sendMessage` internally with the error context.

8. **Abort handling**: Store the AbortController ref. If user sends a new message while one is streaming, abort the previous one.

Return `{ sendMessage, runCypherFromAI, isReady: boolean, providerLabel: string }`.
- `isReady`: true when provider is available (WebGPU check for WebLLM, or API key present for others).
- `providerLabel`: human-readable name of the active provider.
