  <name>Task 1: Create export utility functions with tests</name>
  <files>frontend/src/components/query/export-utils.ts, frontend/src/components/query/export-utils.test.ts</files>
  <behavior>
    - exportAsJson: calls triggerDownload with JSON.stringify of data, application/json MIME type, .json extension
    - exportAsCsv with tabular data (columns + rows present): header row is columns, data rows follow, all cells quoted with double-quote escaping
    - exportAsCsv with graph data (nodes, no columns): header is union of all property keys across nodes prefixed with id and labels columns, each node becomes a row
    - exportAsCsv escapes double quotes by doubling them ("" inside quoted field)
    - exportAsCsv includes BOM prefix (\ufeff) for Excel compatibility
    - exportAsCsv handles nodes with different property sets (missing properties become empty string)
  </behavior>
  <action>
    1. Create frontend/src/components/query/export-utils.ts:
       - Import `QueryResponse` from `@/types/api`

       - `triggerDownload(blob: Blob, filename: string)`: private helper
         - Create URL via `URL.createObjectURL(blob)`
         - Create anchor element, set href and download attributes, click it, revoke URL

       - `exportAsJson(data: QueryResponse, filename = 'query-results.json')`:
         - Create Blob with `JSON.stringify(data, null, 2)`, type `application/json`
         - Call triggerDownload

       - `exportAsCsv(data: QueryResponse, filename = 'query-results.csv')`:
         - If `data.columns && data.rows` (tabular path):
           - Header row: `data.columns`
           - Data rows: `data.rows.map(row => row.map(String))`
         - Else (graph path):
           - Collect union of all property keys: `Array.from(new Set(data.nodes.flatMap(n => Object.keys(n.properties))))`
           - Header: `['id', 'labels', ...allKeys]`
           - Rows: for each node, `[String(n.id), n.labels.join(';'), ...allKeys.map(k => String(n.properties[k] ?? ''))]`
         - CSV encoding: each cell wrapped in double quotes, internal double quotes escaped as `""`, rows joined with `\n`
         - Prepend BOM `\ufeff` for Excel compatibility
         - Create Blob with type `text/csv;charset=utf-8`
         - Call triggerDownload

    2. Create frontend/src/components/query/export-utils.test.ts:
       - Use Node.js built-in test runner (`import { describe, it } from 'node:test'` and `import assert from 'node:assert'`)
       - Follow the existing test pattern from query-utils.test.ts
       - Since triggerDownload uses DOM APIs (document.createElement, URL.createObjectURL), test the CSV and JSON STRING GENERATION logic by extracting or testing the intermediate values
       - Approach: refactor export-utils to have `buildCsvString(data: QueryResponse): string` and `buildJsonString(data: QueryResponse): string` as exported testable functions. The `exportAsCsv` and `exportAsJson` functions call these + triggerDownload.
       - Test cases:
         a. buildJsonString: returns formatted JSON of the QueryResponse
         b. buildCsvString with tabular data: correct header and rows
         c. buildCsvString with graph data: union of property keys, correct header, correct rows
         d. buildCsvString escapes double quotes in values
         e. buildCsvString handles nodes with missing properties (empty cells)
