Existing module pattern (from fraudGraph.ts):
- Each module defines its own local `cloneNode`, `cloneLink`, `cloneGraphData`, and helper functions
- Nodes use string IDs prefixed with a short dataset identifier (e.g., `'fr-acc-1'`)
- Each node has `id`, `labels`, `label` (first label), `properties` with `_label` and domain-specific fields
- The `filterFn` in each GuidedQuery operates on the static data to produce a filtered subgraph
- The `liveDescriptor` field provides the column mapping for live backend mode queries
