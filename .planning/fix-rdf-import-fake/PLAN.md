## (1) Problem summary

`RDFDropzone` presents "Imported N triples · temp db /tmp/playground-xxxxxxxx.ogdb" after a `.ttl` drop. The path is generated by `Math.random()` at `RDFDropzone.tsx:24-27`; no such file exists; no `opengraphdb import-rdf` invocation occurs; no bytes reach disk.

This violates **PLAN D4** from `.planning/playground-premium/PLAN.md`, which required "(a) real write" and explicitly rejected faking. It also invalidates the playground's proof-of-claim role: the landing page markets "bring your ontologies" but the only visible RDF-import surface is a lie.

**Blast radius:** Every playground user who drops an RDF file gets a false success. Discovery of the fake is only possible by inspecting `/tmp/` (empty) or by running `opengraphdb info` against the reported path (ENOENT).

## (5) IMPL SKETCH

Plan adopts option (a) as primary and ships option (b) as the honest fallback when (a)'s preconditions fail.

### Crate: `ogdb-cli` — add `POST /rdf/import`

**File:** `crates/ogdb-cli/src/lib.rs`

1. Extend the HTTP dispatcher `match (request.method.as_str(), request.path.as_str())` (around line 4200, directly above the existing `POST /import`) with a new arm:

   ```rust
   ("POST", path) if path == "/rdf/import" || path.starts_with("/rdf/import?") => {
       let format_hint = resolve_rdf_format_from_http(&request)?;
       let tmp = write_body_to_tempfile(&request.body, format_hint.extension())?;
       let output = handle_import_rdf(
           db_path,
           tmp.path().to_str().unwrap(),
           ImportRdfOptions {
               format_hint: Some(format_hint),
               base_uri: parse_query_param(&request.path, "base_uri"),
               schema_only: false,
               batch_size: DEFAULT_IMPORT_BATCH,
               continue_on_error: false,
               atomic: true,
           },
       ).map_err(|e| CliError::Runtime(e.to_string()))?;
       Ok(http_json_response(200, "OK",
           parse_import_rdf_output_into_json(&output, db_path)?))
   }
   ```

2. **Format resolution order** for the new helper `resolve_rdf_format_from_http`:
   1. `?format=ttl|nt|xml|jsonld|nq|n3` query param
   2. `Content-Type: text/turtle|application/n-triples|application/ld+json|application/n-quads|application/rdf+xml`
   3. Reject with `415 Unsupported Media Type` if neither is present. Do **not** guess from body.

3. `write_body_to_tempfile` uses `std::env::temp_dir()` + a `ogdb-rdf-upload-<uuid>.<ext>` name; drops on `Drop` (RAII). No `tempfile` crate dependency — roll our own 20-line helper (SPEC.md preference for minimal deps). Write via `File::create` + `write_all`.

4. `parse_import_rdf_output_into_json` re-parses `handle_import_rdf`'s `key=value\n` string output into a JSON object. Also include `db_path` (the serve's open path) and a `format` field. Counts must match the CLI.

5. **Size limit:** cap request body at 128 MiB; reject with `413 Payload Too Large`. Derive from new `const RDF_IMPORT_MAX_BYTES: usize = 128 * 1024 * 1024;`.

6. **Auth:** mirror `POST /query`'s bearer-token handling. If `Authorization: Bearer <token>` is present, call `shared_db.authenticate_token`; otherwise `user = "anonymous"`. Write goes through `SharedDatabase::with_write`.

### Frontend: `RDFDropzone.tsx` — wire the real POST

**File:** `frontend/src/components/playground/RDFDropzone.tsx`

1. Remove `tempDbPath()` (the `Math.random` invention).
2. Introduce state variants: `'uploading'`, `'persisted'`, `'preview-only'`.
3. Add `async function uploadRdf(file: File, format: string): Promise<ImportResponse | { kind: 'backend-down' }>`:
   - `fetch('/api/rdf/import?format=' + format, { method: 'POST', body: await file.text(), headers: { 'Content-Type': contentTypeFor(format) } })`
   - On **ok**: `ImportResponse` with `db_path`, `imported_nodes`, `imported_edges`, etc.
   - On **502 from vite / ECONNREFUSED / 500 from proxy**: return `{ kind: 'backend-down' }`.
   - On other **4xx/5xx**: throw, surface as `phase: 'error'` with server's error message.
4. `handleCommit` flow:
   - Start `phase: 'uploading'`.
   - Call `uploadRdf`.
   - If `ok` → `phase: 'persisted'`, surface *live db_path + counts from backend*, update canvas via `triplesToGraphData` (parsed client-side for immediate visual; canonical truth is now on server, UI can later re-fetch `/api/schema`).
   - If `backend-down` → `phase: 'preview-only'`, show yellow pill: *"Preview only · not persisted — start `ogdb serve --http` to ingest for real"*. Still swap the canvas (as today) but drop the fake path entirely. Button copy changes to "Preview in canvas".
5. New `data-testid` hooks: `rdf-import-persisted`, `rdf-import-preview-only`, `rdf-import-db-path`, `rdf-import-count-total`.
6. Remove the string *"Commits to temp .ogdb · preview only, not persisted"* — it is misleading in both old and new flows. Replace with state-specific copy.

### Frontend: `frontend/src/lib/rdfClient.ts` (new, tiny)

Extract `uploadRdf` + `contentTypeFor` + `ImportResponse` type here so the dropzone stays UI-only and the unit tests (`test:unit`) can cover the client without JSDOM gymnastics.

### Propagation

- `PlaygroundPage.tsx:onImport` gains a third arg `source: 'live' | 'preview'` (or fold into the existing `source` object). Canvas renders normally; `StatusBar` optionally surfaces the live db path.
- No changes to `ogdb-core`, `ogdb-bolt`, `ogdb-ffi`, `ogdb-node`, `ogdb-python`. Strictly an HTTP + UI change.

### Out of scope for this phase — called out so we don't sneak it in

- **Auto-spawning `ogdb serve --http` from vite dev middleware.** Interesting; requires vite plugin + cargo detection + process lifecycle. Ship as a follow-up phase. For now the user runs it themselves; the UI is honest about when it isn't running.
- **Per-drop fresh `.ogdb` files.** The server is serving one open DB. Multi-DB tenancy is a separate design; don't bolt on here.
- **Multipart form-data.** We take raw body + Content-Type/query format. Simpler, matches `ogdb` CLI's single-path model.
- **RDF validation / SHACL gating.** Already available as `opengraphdb validate` post-import; not blocking this fix.

## (7) DECISION LOG

| # | Decision | Alternatives considered | Why this choice |
|---|---|---|---|
| D1 | New endpoint is `POST /rdf/import` (not repurpose `POST /import`) | Extending `/import` to detect RDF by `Content-Type` | Keeps `/import` semantics (records array) stable; RDF has its own format negotiation, size limits, error model. Cleaner API. |
| D2 | Raw body + `Content-Type`/`?format=`, not multipart | `multipart/form-data` | Simpler parser (no existing multipart code in `ogdb-cli`), matches `handle_import_rdf`'s single-path shape, cheaper tests. |
| D3 | Server writes to tempfile, then calls `handle_import_rdf(path)` | Refactor `handle_import_rdf` to accept `impl Read` | Less surface area to change this phase; temp-file cost is negligible (bounded 128 MiB, `/tmp` is tmpfs). Keeps CLI and HTTP paths converging on the same code. |
| D4 | 128 MiB body cap | unlimited / 1 GiB / 10 MiB | Matches rough Turtle-file expectations for playground use; larger = user runs CLI directly. |
| D5 | Auth optional (Bearer token if present, else anonymous) | Always require auth | Mirrors existing `POST /query`; avoids regressing the playground dev path where the CLI is launched without auth. |
| D6 | Frontend keeps client-side parse for preview counts | Delete client parse, show spinner until server responds | Preview counts give the user confidence before they commit; deleting them slows the UX without user benefit. |
| D7 | Fallback UI ("preview only") when backend 502/ECONNREFUSED | Hard-fail with error | PLAN D4 rejected silent fakes; a loud "not persisted" pill is the honest middle ground — user sees canvas preview but is told unambiguously nothing was saved. |
| D8 | No vite auto-spawn this phase | Ship vite plugin now | Scope creep; cargo/Rust availability is an end-user concern (docs), not a dev-server concern. Punt to a follow-up. |
| D9 | Per-crate cargo only, as instructed | Workspace builds | Repo is 10 crates; workspace builds are slow and this change is strictly `ogdb-cli` + frontend. |
| D10 | RED test spawns backend via `cargo run -p ogdb-cli` in `beforeAll` | Require user to prestart backend | Self-contained e2e; CI needs zero manual steps. Gracefully `test.skip`s if cargo absent. |
