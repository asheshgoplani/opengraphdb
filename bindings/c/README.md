# bindings/c

C foreign-function-interface header for OpenGraphDB. The header
(`opengraphdb.h`) is **auto-generated by `cbindgen.toml`** at
workspace root from `crates/ogdb-ffi`. Do not hand-edit; the next
regenerate will overwrite changes.

## Status

Experimental. The C surface is stable per the workspace `v0.x`
semver guarantee — see `crates/ogdb-ffi/src/lib.rs` `//!` block
for the full surface.

## Regenerate the header

After a change to `crates/ogdb-ffi`:

```bash
cargo install cbindgen --locked
cbindgen --config cbindgen.toml --crate ogdb-ffi --output bindings/c/opengraphdb.h
```

The header's contents are pinned by `[export].include` in
`cbindgen.toml`; adding a new pub `extern "C" fn` to ogdb-ffi
without listing it there silently omits it from the regenerate.

## Build the example

`example.c` is a minimal walkthrough that initialises a database,
creates two nodes, adds an edge, runs a Cypher query, and frees
the result.

```bash
cargo build --release -p ogdb-ffi
gcc -I bindings/c -L target/release -logdb_ffi -ldl -lpthread \
  bindings/c/example.c -o example
LD_LIBRARY_PATH=target/release ./example
```

On macOS the linker incantation is the same with `-ldl -lpthread`
elided and `DYLD_LIBRARY_PATH` substituted for `LD_LIBRARY_PATH`.

## Memory ownership

Strings returned by `ogdb_query` / `ogdb_metrics` are heap-allocated
on the Rust side; release them with `ogdb_free` after you finish
reading. `ogdb_close` takes ownership of the handle and invalidates
it — do not call any other `ogdb_*` function on a closed handle.
