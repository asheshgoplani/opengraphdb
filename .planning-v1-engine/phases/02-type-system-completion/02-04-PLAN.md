NOTE on disambiguation: In primary position, `{` starts a MapLiteral. In postfix position (after an expression), `{` starts a MapProjection. The postfix parser must only consume `{` after confirming it contains identifier-only entries (no colons), to avoid swallowing a MapLiteral that belongs to the next statement. However, since map projection in postfix position only fires when `{` immediately follows an expression (inside the postfix loop), and a primary `{` is only reached from `parse_primary_expression`, the two paths don't conflict. The postfix loop should check for `{` and peek to confirm it's a projection (identifiers without colons).

**Step 13: Extend parse_postfix_expression for map projection**

In the postfix loop (after the `LeftBracket` handling from 02-03), add a check for `LeftBrace` that disambiguates map projection from other uses of `{`:

```rust
} else if self.check_punctuation(CypherPunctuation::LeftBrace) {
    // Map projection: expr{key1, key2, ...}
    // Peek ahead to disambiguate from a map literal in the next statement.
    // If the tokens inside braces are only identifiers separated by commas (no colons),
    // this is a map projection.
    let checkpoint = self.index;
    self.expect_punctuation(CypherPunctuation::LeftBrace).unwrap();
    let mut is_projection = true;
    let mut keys = Vec::new();

    // Try to parse as projection (identifier list without colons)
    if self.check_identifier() {
        let first_key = self.parse_identifier("for map projection key").unwrap();
        keys.push(first_key);
        while self.match_punctuation(CypherPunctuation::Comma) {
            if self.check_identifier() {
                let key = self.parse_identifier("for map projection key").unwrap();
                keys.push(key);
            } else {
                is_projection = false;
                break;
            }
        }
        if is_projection && self.match_punctuation(CypherPunctuation::RightBrace) {
            expr = CypherExpression::MapProjection {
                base: Box::new(expr),
                keys,
            };
            continue; // continue the postfix loop
        }
    }
    // Not a map projection; backtrack
    self.index = checkpoint;
    break;
}
```

**IMPORTANT:** This uses a checkpoint/backtrack pattern. The parser must support resetting `self.index` (which it does, as seen in `parse_list_expression` at line ~6080 where `let checkpoint = self.index;` and `self.index = checkpoint;` are used).

**Step 2: Add MapProjection evaluator**

Add a new match arm in `evaluate_expression` for the MapProjection variant:

```rust
CypherExpression::MapProjection { base, keys } => {
    let base_val = self.evaluate_expression(base, row, snapshot_txn_id)?;
    match base_val {
        RuntimeValue::Property(PropertyValue::Map(map)) => {
            let projected: BTreeMap<String, PropertyValue> = keys.iter()
                .filter_map(|key| map.get(key).map(|v| (key.clone(), v.clone())))
                .collect();
            Ok(RuntimeValue::Property(PropertyValue::Map(projected)))
        }
        RuntimeValue::Node(node_id) => {
            // Map projection on a node: returns a map of selected properties
            let props = self.node_properties_at(node_id, snapshot_txn_id)?;
            let projected: BTreeMap<String, PropertyValue> = keys.iter()
                .filter_map(|key| props.get(key).map(|v| (key.clone(), v.clone())))
                .collect();
            Ok(RuntimeValue::Property(PropertyValue::Map(projected)))
        }
        RuntimeValue::Edge(edge_ref) => {
            // Map projection on an edge: returns a map of selected properties
            let props = self.edge_properties_at(edge_ref.edge_id, snapshot_txn_id)?;
            let projected: BTreeMap<String, PropertyValue> = keys.iter()
                .filter_map(|key| props.get(key).map(|v| (key.clone(), v.clone())))
                .collect();
            Ok(RuntimeValue::Property(PropertyValue::Map(projected)))
        }
        _ => Ok(RuntimeValue::Null),
    }
}
```
