<objective>
Add factorized intermediate result representation to the OpenGraphDB execution engine to avoid Cartesian explosion on multi-pattern queries.

Purpose: QOPT-02 requires the execution engine to support factorized intermediate results. The current execution model fully materializes the Cartesian product at each operator boundary. For queries like `MATCH (a)-[:R]->(b), (a)-[:S]->(c) RETURN a, b, c`, if node `a` has 1000 R-neighbors and 1000 S-neighbors, the binary approach produces 1,000,000 intermediate rows, while a factorized representation stores only 2000 values plus the tree structure.

Output: `FactorTree` and `FactorNode` structs, `PhysicalFactorizedExpand` plan variant, factorized execution in the expand operator, `materialize_factor_tree()` for final output, and fan-out based selection in the physical planner.
</objective>

/// A node in a factorized result tree.
///
/// Each FactorNode represents a variable binding. Children represent dependent
/// variable bindings that are valid for this particular value.
///
/// Example: for MATCH (a)-[:R]->(b), (a)-[:S]->(c):
/// ```text
/// FactorNode { variable: "a", value: Node(0), children: [
///     FactorNode { variable: "b", value: Node(1), children: [] },
///     FactorNode { variable: "b", value: Node(2), children: [] },
///     FactorNode { variable: "c", value: Node(3), children: [] },
///     FactorNode { variable: "c", value: Node(4), children: [] },
/// ]}
/// ```
///
/// The factorized size is O(|b| + |c|) per `a` value, while flat would be O(|b| * |c|).
#[derive(Debug, Clone)]
struct FactorNode {
    /// Variable name this node binds.
    variable: String,
    /// The bound value.
    value: RuntimeValue,
    /// Optional edge reference (if this binding came from an edge traversal).
    edge_variable: Option<String>,
    /// Optional edge value.
    edge_value: Option<RuntimeValue>,
    /// Child bindings dependent on this value.
    children: Vec<FactorNode>,
}

/// A factorized intermediate result tree.
///
/// Instead of materializing the full Cartesian product of all variable bindings,
/// this tree stores bindings hierarchically. The full result set is recovered by
/// computing the Cartesian product of independent subtrees during materialization.
///
/// A FactorTree has a list of root-level variable groups. Variables within the same
/// group are correlated (connected by edges). Independent groups are combined via
/// Cartesian product only at materialization time.
#[derive(Debug, Clone)]
struct FactorTree {
    /// Variable groups, each a list of FactorNode roots.
    /// Independent groups are cross-producted during materialization.
    groups: Vec<FactorGroup>,
}

/// A group of correlated variable bindings in the factor tree.
#[derive(Debug, Clone)]
struct FactorGroup {
    /// The top-level variable name for this group.
    variable: String,
    /// Root nodes: each root has a value for `variable` and children for dependent variables.
    roots: Vec<FactorNode>,
    /// Names of all variables present in this group (including nested).
    all_variables: Vec<String>,
}
```

**Step 2: Add materialize_factor_tree() function**

Add this function near the `rows_from_batches` and `batches_from_rows` helpers (around line ~5300):

```rust
/// Materialize a FactorTree into flat rows.
///
/// For each group in the tree, collect all complete variable bindings.
/// Then compute the Cartesian product across independent groups.
fn materialize_factor_tree(tree: &FactorTree) -> Vec<BTreeMap<String, RuntimeValue>> {
    if tree.groups.is_empty() {
        return Vec::new();
    }

    // Materialize each group into a list of partial rows.
    let mut group_rows: Vec<Vec<BTreeMap<String, RuntimeValue>>> = Vec::new();

    for group in &tree.groups {
        let mut rows = Vec::new();
        for root in &group.roots {
            let mut partial = BTreeMap::new();
            materialize_factor_node(root, &mut partial, &mut rows);
        }
        if rows.is_empty() {
            // If any group has no rows, the Cartesian product is empty.
            return Vec::new();
        }
        group_rows.push(rows);
    }

    // Compute Cartesian product across independent groups.
    if group_rows.len() == 1 {
        return group_rows.into_iter().next().unwrap();
    }

    let mut result = group_rows[0].clone();
    for group_idx in 1..group_rows.len() {
        let right = &group_rows[group_idx];
        let mut new_result = Vec::with_capacity(result.len() * right.len());
        for left_row in &result {
            for right_row in right {
                let mut merged = left_row.clone();
                for (key, value) in right_row {
                    merged.insert(key.clone(), value.clone());
                }
                new_result.push(merged);
            }
        }
        result = new_result;
    }

    result
}

/// Recursively materialize a FactorNode into partial row bindings.
/// When a leaf node is reached (no children), emit the accumulated partial row.
fn materialize_factor_node(
    node: &FactorNode,
    partial: &mut BTreeMap<String, RuntimeValue>,
    output: &mut Vec<BTreeMap<String, RuntimeValue>>,
) {
    // Bind this variable.
    partial.insert(node.variable.clone(), node.value.clone());

    // Bind edge variable if present.
    if let (Some(ref ev), Some(ref edge_val)) = (&node.edge_variable, &node.edge_value) {
        partial.insert(ev.clone(), edge_val.clone());
    }

    if node.children.is_empty() {
        // Leaf: emit the current partial row.
        output.push(partial.clone());
    } else {
        // Group children by variable name to handle independent subtrees.
        let mut children_by_var: BTreeMap<String, Vec<&FactorNode>> = BTreeMap::new();
        for child in &node.children {
            children_by_var
                .entry(child.variable.clone())
                .or_default()
                .push(child);
        }

        if children_by_var.len() == 1 {
            // All children bind the same variable: iterate over them.
            for child in &node.children {
                materialize_factor_node(child, partial, output);
            }
        } else {
            // Multiple independent variables: factorized Cartesian product.
            let mut var_rows: Vec<Vec<BTreeMap<String, RuntimeValue>>> = Vec::new();
            for (_var, children) in &children_by_var {
                let mut rows = Vec::new();
                for child in children {
                    let mut child_partial = BTreeMap::new();
                    materialize_factor_node(child, &mut child_partial, &mut rows);
                }
                var_rows.push(rows);
            }

            // Cross-product the independent variable groups.
            let mut combined = var_rows[0].clone();
            for group_idx in 1..var_rows.len() {
                let right = &var_rows[group_idx];
                let mut new_combined = Vec::with_capacity(combined.len() * right.len());
                for left in &combined {
                    for right_row in right {
                        let mut merged = left.clone();
                        for (k, v) in right_row {
                            merged.insert(k.clone(), v.clone());
                        }
                        new_combined.push(merged);
                    }
                }
                combined = new_combined;
            }

            // Merge each combined row with the current partial.
            for combined_row in combined {
                let mut full = partial.clone();
                for (k, v) in combined_row {
                    full.insert(k, v);
                }
                output.push(full);
            }
        }
    }

    // Unbind for backtracking.
    partial.remove(&node.variable);
    if let Some(ref ev) = node.edge_variable {
        partial.remove(ev);
    }
}
```

                    // We need to pair each materialized row with its corresponding input row.
                    // The factor tree root index maps back to the input row.
                    // For simplicity, re-expand from input rows using the lookup.
                    std::iter::repeat(()).take(materialized.len())
                ) {
                    final_rows.push(mat_row.clone());
                }

                // Rebuild complete rows: the factor tree only stored from_var and to_var.
                // We need to re-attach all input columns.
                let mut complete_rows = Vec::<BTreeMap<String, RuntimeValue>>::new();
                for input_row in &input_rows {
                    let src = match input_row.get(from) {
                        Some(RuntimeValue::Node(value)) => *value,
                        Some(RuntimeValue::Property(PropertyValue::I64(value))) => *value as u64,
                        _ => continue,
                    };
                    let matches = lookup.get(&src).cloned().unwrap_or_default();
                    for (edge_ref, neighbor) in matches {
                        let mut out_row = input_row.clone();
                        if let Some(ev) = edge_variable {
                            out_row.insert(ev.clone(), RuntimeValue::Edge(edge_ref));
                        }
                        out_row.insert(to_var.clone(), RuntimeValue::Node(neighbor));
                        complete_rows.push(out_row);
                    }
                }

                Ok(batches_from_rows(&complete_rows, &output_columns, QUERY_BATCH_SIZE))
            }
```

**IMPORTANT NOTE:** The factorized execution path above builds a FactorTree and then materializes it. For a single expansion step, the savings come when **multiple** factorized expansions are chained (the tree grows deeper without materializing intermediate Cartesian products). For this initial implementation, the key value is:

1. The data structures (FactorTree, FactorNode, FactorGroup) are established.
2. The planner selects factorized execution based on fan-out.
3. The materialization logic correctly produces the same rows as flat execution.
4. Future work can chain multiple factorized expansions without intermediate materialization.

The current executor materializes at each operator boundary (same as flat), but the FactorTree infrastructure is in place for future lazy materialization across operator chains.
